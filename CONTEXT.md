# Hoppy

Hoppy(구 dev-screen)은 웹 프론트엔드 개발자가 개발한 화면을 녹화하고 자동 효과(줌, 커서 스무딩)를 입혀, GitHub issue/PR과 업무 티켓에 첨부할 짧은 데모 영상(1~2분)을 만드는 macOS 앱. Screen Studio와 Cap의 공통 코어 기능만 벤치마킹한다.

## Language

**녹화 (Recording)**:
캡처된 원본 화면 영상과 이벤트 트랙의 묶음. 효과가 입혀지지 않은 원천 데이터.
_Avoid_: 영상, 비디오 (최종 산출물과 혼동됨)

**이벤트 트랙 (Event Track)**:
녹화 중 별도로 기록되는 마우스 이벤트 로그(위치, 클릭, 타임스탬프). 원본 영상과 분리 저장되며 자동 효과 생성의 입력이 된다.

**렌더 레시피 (Render Recipe)**:
녹화를 최종 영상으로 합성하는 데 필요한 파라미터 전체 — 줌 구간 목록, 트림 지점, 배경/패딩 스타일, 커서 설정. 편집이란 이 레시피를 고치는 행위다.
_Avoid_: 프로젝트 설정, 편집 상태

**줌 구간 (Zoom Segment)**:
타임라인상 특정 시간 범위에 적용되는 확대 효과 단위. 클릭 이벤트로부터 자동 생성되며, 사용자가 삭제/이동/길이 조절할 수 있다.

**팬 (Pan)**:
줌인 상태를 유지한 채 카메라 중심만 이동하는 것. 확대 중 다음 클릭이 현재 뷰 밖에서 일어나면 줌아웃 대신 팬으로 연결한다.

**자동 효과 (Auto Effects)**:
이벤트 트랙으로부터 자동 유도되는 효과들 — 클릭 기반 자동 줌, 커서 스무딩, 클릭 하이라이트. 녹화 중이 아니라 후처리에서 적용된다.

**경량 편집 (Light Edit)**:
v1이 허용하는 편집의 전부: 앞뒤 트리밍, 줌 구간 조정, 배경/패딩 스타일 조절. 컷 편집(중간 잘라내기)·속도 조절·자막은 명시적으로 제외 — 실수하면 재녹화한다.

**익스포트 (Export)**:
렌더 레시피를 원본에 적용해 최종 파일(MP4/GIF)을 만드는 후처리 렌더링. 녹화 종료와 동시에 일어나지 않고 미리보기 확인 후 실행된다.

**익스포트 프리셋 (Export Preset)**:
목적지의 첨부 제한에 맞춰 포맷·해상도·비트레이트를 묶은 익스포트 설정. v1은 GitHub 프리셋(이미지/GIF 10MB, 영상 100MB 제한 타겟) 하나로 시작한다.

**온보딩 (Onboarding)**:
최초 실행 시 1회 진행되는 안내 흐름 — 권한 확보(화면 녹화·손쉬운 사용)와 기능 소개. 완료 여부는 로컬에 저장되며, 완료 후 권한이 회수되면 온보딩이 아니라 녹화 시점의 오류 안내가 처리한다.
_Avoid_: 권한 게이트 (권한 확보는 온보딩의 한 단계일 뿐, 전체가 아님)

**오버레이 배지 (Overlay Badge)**:
최종 영상에 합성되는 컨텍스트 정보 요소. v1은 뷰포트 크기 배지(예: 1440×900), v2에서 브랜치/커밋 배지·키 입력 오버레이가 추가된다. 렌더 레시피의 일부다.
