# Recap

Recap(구 dev-screen)은 웹 프론트엔드 개발자가 개발한 화면을 녹화하고 자동 효과(줌, 커서 스무딩)를 입혀, NHN Dooray 업무에 첨부할 짧은 데모 GIF(1~2분)를 만드는 macOS 앱. Screen Studio와 Cap의 공통 코어 기능만 벤치마킹한다.

## Language

**녹화 (Recording)**:
캡처된 원본 화면 영상과 이벤트 트랙의 묶음. 효과가 입혀지지 않은 원천 데이터.
_Avoid_: 영상, 비디오 (최종 산출물과 혼동됨)

**이벤트 트랙 (Event Track)**:
녹화 중 별도로 기록되는 마우스 이벤트 로그(위치, 클릭, 타임스탬프). 원본 영상과 분리 저장되며 자동 효과 생성의 입력이 된다.

**렌더 레시피 (Render Recipe)**:
녹화를 최종 영상으로 합성하는 데 필요한 파라미터 전체 — 클립 시퀀스, 줌 구간 목록, 배경/패딩 스타일, 커서 설정. 편집이란 이 레시피를 고치는 행위다. 트림은 이제 클립 시퀀스의 양끝 경계로 표현된다.
_Avoid_: 프로젝트 설정, 편집 상태

**클립 (Clip)**:
렌더 레시피가 최종 영상으로 남기는 원본의 한 조각. `[sourceStartMs, sourceEndMs]` 구간과 재생 속도 배율을 가진다. 컷/분할은 클립을 나누고, 속도 조절은 클립의 배율을 바꾼다.
_Avoid_: 슬라이스(내부 표현), 세그먼트(줌 구간과 혼동)

**클립 시퀀스 (Clip Sequence)**:
source 오름차순·비겹침으로 늘어선 클립들의 순서 목록. 최종 영상의 출력 타임라인은 이 클립들을 이어 붙인 것이며, 클립 사이 간극이 컷(제거된 구간)이다. 출력 시간↔원본 시간 매핑의 입력이다.

**줌 구간 (Zoom Segment)**:
타임라인상 특정 시간 범위에 적용되는 확대 효과 단위. 클릭 이벤트로부터 자동 생성되며, 사용자가 삭제/이동/길이 조절할 수 있다. 원본(source) 시간에 앵커되며, 컷/속도는 별도 매핑 계층이 흡수한다.

**팬 (Pan)**:
줌인 상태를 유지한 채 카메라 중심만 이동하는 것. 확대 중 다음 클릭이 현재 뷰 밖에서 일어나면 줌아웃 대신 팬으로 연결한다.

**자동 효과 (Auto Effects)**:
이벤트 트랙으로부터 자동 유도되는 효과들 — 클릭 기반 자동 줌, 커서 스무딩, 클릭 하이라이트. 녹화 중이 아니라 후처리에서 적용된다.

**경량 편집 (Light Edit)**:
허용하는 편집: 앞뒤 트리밍, 컷(분할·삭제), 속도 조절, 줌 구간 조정, 배경/패딩 스타일 조절. 자막은 계속 제외한다. 트림·클립·줌 범위를 넘어선 편집은 재녹화로 해결한다.

**익스포트 (Export)**:
렌더 레시피를 원본에 적용해 최종 GIF를 만드는 후처리 렌더링. 녹화 종료와 동시에 일어나지 않고 미리보기 확인 후 실행된다. export 출력은 GIF 단일이다(녹화 소스는 계속 MP4).

**익스포트 프리셋 (Export Preset)**:
목적지(Dooray 업무 첨부)에 맞춰 GIF 색 수·용량 경고 정책을 묶은 익스포트 설정. v1은 Dooray GIF 프리셋 하나로 시작한다. 해상도·프레임레이트는 프리셋이 아니라 익스포트 시점의 사용자 선택(GifSelection — 드롭다운)이며, 원본이 상한이다(업스케일 금지). #122

**온보딩 (Onboarding)**:
최초 실행 시 1회 진행되는 안내 흐름 — 권한 확보(화면 녹화·손쉬운 사용)와 기능 소개. 완료 여부는 로컬에 저장되며, 완료 후 권한이 회수되면 온보딩이 아니라 녹화 시점의 오류 안내가 처리한다.
_Avoid_: 권한 게이트 (권한 확보는 온보딩의 한 단계일 뿐, 전체가 아님)

**오버레이 배지 (Overlay Badge)**:
최종 영상에 합성되는 컨텍스트 정보 요소. v1은 뷰포트 크기 배지(예: 1440×900), v2에서 브랜치/커밋 배지·키 입력 오버레이가 추가된다. 렌더 레시피의 일부다.
